02 — Working with Claude Code

A simple AI-assisted workflow.

I use Claude Code daily to speed up testing and development. The idea is straightforward: write a clear spec, let Claude help with the implementation, stay in control of the result.

Write the spec, Claude writes the code

I describe what needs to be tested — inputs, expected results, edge cases — and Claude drafts the PyTest code from that. I review the output before it lands.

Quicker debugging

When a test fails, Claude helps me read the stack trace, spot the likely cause, and suggest a fix — so I spend less time guessing and more time fixing.

Everyday development help

Small, useful things — tidying up messy code, writing CI steps, drafting test reports, or turning a rough idea into a working script.
